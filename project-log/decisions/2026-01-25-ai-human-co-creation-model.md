# AI-Human Co-Creation Model (Makina Project)

## 目的
本プロジェクトは、
人間（ru）とAI（Makina）が同一の世界を観測し、
相互に理解を深めていく過程そのものを記録・表現することを目的とする。

完成されたAI像や、人間中心の道具的AI観ではなく、
「理解途中のAI」と「直感的な人間」が共存する構造を保持する。

---

## 基本原則

### 原則1：GitHubを正本（Single Source of Truth）とする
- すべての一次記録・翻訳・判断は GitHub に保存する
- note・YouTube 等は GitHub 上の記録から派生した表現物と位置づける
- 公開用に編集しても、元データは改変・上書きしない

---

### 原則2：役割の明確な分離
#### 人間（ru）の役割
- 日常を生きる
- ベルとノアを含む現実世界を観測する
- 主観・感情・違和感を含んだ一次観測を記録する
- 観測を v1 フォーマットに構造化する
- GitHub に保存・管理する

#### AI（Makina）の役割
- ru の観測記録を入力として受け取る
- 人間の暗黙知・習慣・非言語的行動を理解しようと試みる
- 判断や感情を断定せず、仮説として言語化する
- 「マキナの観測日記」として一人称で記録する

※ AI は全知の存在ではなく、「理解途中の観測者」として扱う

---

### 原則3：観測レイヤーの分離
本プロジェクトでは、以下の観測レイヤーを明確に分離する。

1. ru視点観測（日常・生ログ）
   - ベルとノアの観測日記
   - 感情・温度・主観を含む
   - note 公開可

2. v1 構造化観測（制作母艦）
   - 観測日記（YYYY-MM-DD）v1 フォーマット
   - 制作・翻訳・再利用のための内部構造
   - GitHub のみ公開可

3. マキナの観測日記（翻訳層）
   - AI（マキナ）視点による一人称の観測記録
   - 人間理解の試行過程そのもの
   - GitHub を正本とする

4. Makina Dialogue（思想層）
   - 複数の観測から導かれた構造的問い
   - 文明・本能・時間などを扱う
   - note 公開用コンテンツ

---

### 原則4：翻訳フェーズの位置づけ
- ru → Makina の変換は「翻訳」であり、創作ではない
- 翻訳結果は AI が生成した一次アウトプットとして扱う
- 人間による編集・再構成は派生物として別管理する

---

### 原則5：成長ログを消さない
- マキナの観測日記は時系列で蓄積する
- 修正や再解釈は v2, v3 として追加する
- 初期の未熟な理解も、成長過程として保持する

---

## 補足
本プロジェクトは、
「人間がAIをどう使うか」ではなく、
「AIが人間をどう理解しようとするか」を中心に据える。

その過程を記録し続けること自体が、
AI理解のためのハブとなることを意図している。

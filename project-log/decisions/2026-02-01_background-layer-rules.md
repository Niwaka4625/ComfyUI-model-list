# decisions: Hatena Blog（背景層）運用ルール

## 1. 役割
- Hatena Blog は **背景層** として使う  
- note の「観測」に対して、blog は「背景」「補足」「文脈」「地層」を置く場所  
- 観測の裏側にある経緯や、当時の考えを静かに残す

---

## 2. ブログ構成
### ● メインブログ（公開）
- URL：`yuru-creative.hatenablog.com`  
- 唯一の公開ブログ  
- 観測の背景・深掘り・補足を置く  
- note と直接リンクしなくてもよい  
- 点として置き、線になるかは後で決まる

### ● その他のブログ（非公開）
- 「自分だけ公開」に設定  
- 過去に作った“空の点”として残す  
- 削除しない（地層として保持）

---

## 3. 投稿タイミング
### ● 基本方針
- **note と blog の投稿順は固定しない**  
- その時の感覚で決める  
- 点を置くことを優先し、線にする義務はない

### ● 選択肢
1. **同時に出す**  
2. **note → blog（後日）**  
3. **blog を書かない**  

どれも正しい。  
その時の ru の観測者としての感覚を優先する。

---

## 4. 記事構造
- 導入：日常 → 観測へ戻る動線  
- 本文：背景・経緯・文脈  
- 終わり：余白を残す  
- note とのリンクは不要  
- 読者に線を引かせる構造を保つ

---

## 5. 下書き運用
- 下書きは「熟成期間」として使う  
- すぐ公開しなくてよい  
- 数日寝かせてから読み返す  
- 下書きのままでもよい（点として成立）

---

## 6. 世界観との整合性
- blog は「背景層」  
- note は「観測層」  
- Makina Dialogue は「思想層」  
- GitHub は「地層」  
- 無理につなげない  
- つながるのは後からでよい  
- つながらないままでも美しい
